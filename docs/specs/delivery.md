# Delivery

How we hand the build off — prototype, accessibility verification, runbooks, ownership.

## Prototype deliverables
- Figma file with all ~95 screens (light + dark + RTL + Android variants) generated by the plugin
- Make app demoing the consumer + admin shells with mock data
- Plugin source repo: github.com/ktech643/figma_roam_plugin
- Spec docs: `docs/specs/` (this directory)
- Master config sample: `config/secrets.env.example`

## Handoff package per surface
| Surface | Repo | Build | Deploy |
|---------|------|-------|--------|
| Backend | `roamlu/api` | `pnpm build` → Docker | ECS Fargate |
| Web admin | `roamlu/admin` | `pnpm build` → static | Cloudflare Pages |
| Mobile | `roamlu/mobile` | Flutter → IPA/AAB | TestFlight + Play |
| Plugin | `figma_roam_plugin` | bundled JS | Figma plugin manifest |

## Accessibility verification
- Mobile: VoiceOver (iOS) + TalkBack (Android) full pass per surface
- Web: axe-core in CI + manual NVDA pass on A00–A19
- Color contrast: WCAG AA minimum, AAA for body text
- Hit targets: 44pt minimum (iOS), 48dp (Android)
- Reduced motion respected for all transitions
- RTL: every screen mirrored; numbers stay LTR

## Runbooks (must exist before launch)
1. Webhook backlog (B5)
2. Provisioning queue stuck (B7)
3. Reconciliation mismatch (B6 / FL)
4. Auth incident (token leak, brute force)
5. Provider outage (Stripe / Airalo / Twilio / WireGuard)
6. Region flag toggle (legal/regulatory)
7. ZATCA submission failure
8. Database failover
9. Force-update rollout
10. Account-deletion exception handling
